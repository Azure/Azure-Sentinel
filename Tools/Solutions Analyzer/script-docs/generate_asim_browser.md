# ASIM Schema Browser Generator

**Script:** `generate_asim_browser.py`

## Overview

Generates an interactive single-page HTML application (`asim-browser.html`) for browsing ASIM schema field definitions and parsers. The page has:

- **Primary tabs** for each ASIM schema (Authentication, Dns, NetworkSession, etc.)
- **Sub-tabs** within each schema:
  - **Fields** — Interactive DataTable with all field definitions, classes, types, and source indicators
  - **Parsers** — Interactive DataTable listing unifying and source parsers for the schema

The visual style matches the main `index.html` generated by `generate_interactive_docs.py`.

## Prerequisites

- Python 3.7 or higher
- Pre-generated input CSVs:
  - `asim_fields.csv` (from `collect_asim_fields.py`)
  - `asim_parsers.csv` (from `map_solutions_connectors_tables.py`)

No additional Python packages are required beyond the standard library.

## Running the Script

From the `Tools/Solutions Analyzer` directory:

```bash
# Generate locally for testing (links won't resolve)
python generate_asim_browser.py

# Generate for GitHub Pages deployment (sentinelninja repo)
python generate_asim_browser.py \
    --output-dir "C:\Users\ofshezaf\GitHub\sentinelninja" \
    --docs-base-path "Solutions Docs/" \
    --index-url "https://oshezaf.github.io/sentinelninja/index.html" \
    --link-extension ".html"
```

## Command Line Options

| Option | Default | Description |
|--------|---------|-------------|
| `--fields-csv` | `asim_fields.csv` in script directory | Path to asim_fields.csv |
| `--parsers-csv` | `asim_parsers.csv` in script directory | Path to asim_parsers.csv |
| `--output-dir` | Script directory | Directory to write `asim-browser.html` into |
| `--docs-base-path` | `""` (empty) | Relative path prefix from asim-browser.html to entity doc pages (e.g., `"Solutions Docs/"`) |
| `--index-url` | `""` (empty) | URL to the main index.html for the navbar link |
| `--link-extension` | `.html` | File extension for entity page links (`.html` or `.md`) |

## Output

| File | Description |
|------|-------------|
| `asim-browser.html` | Self-contained HTML page with inline CSS and JavaScript |

The page is fully self-contained — all CSS and JavaScript are inline (no separate `css/` or `js/` files needed). External CDN resources (Bootstrap 5, jQuery, DataTables.js) are loaded at runtime.

## How It Works

### Data Flow

```
┌──────────────────────┐    ┌──────────────────────┐
│   asim_fields.csv    │    │  asim_parsers.csv    │
│ (collect_asim_fields)│    │ (mapper script)      │
└──────────┬───────────┘    └──────────┬───────────┘
           │                           │
           ▼                           ▼
┌─────────────────────────────────────────────────┐
│          generate_asim_browser.py                │
│                                                   │
│  • Group fields and parsers by schema            │
│  • Generate HTML with schema tabs + sub-tabs     │
│  • Inline CSS/JS matching interactive index style│
│  • Build DataTable for fields (filterable)       │
│  • Build DataTable for parsers (filterable)      │
│  • Generate entity links using docs-base-path    │
└──────────────────────────┬──────────────────────┘
                           │
                           ▼
                   asim-browser.html
```

### Page Structure

The generated page uses Bootstrap 5 tabs and DataTables.js:

1. **Navbar** — Links to the main Solutions Analyzer index and ASIM documentation
2. **Summary cards** — Schema count, total fields, common fields, total/source parsers
3. **Schema tabs** — One tab per schema (11 schemas), ordered by category relevance
4. **Fields sub-tab** — Interactive DataTable with per-column filters showing:
   - Field name, class, physical type, logical type
   - Source (Schema, Common, Tester Only, Physical Table Only)
   - Entity, role, description
   - In Tester (✓/—), In Physical Table (✓/—)
5. **Parsers sub-tab** — Interactive DataTable showing:
   - Parser name (linked to parser doc page), type (Unifying/Source)
   - Product, version, tables, solutions (linked to solution doc pages)

### Schema Tab Order

Schemas are displayed in this fixed order:

1. Authentication
2. Dns
3. NetworkSession
4. WebSession
5. FileEvent
6. ProcessEvent
7. RegistryEvent
8. AuditEvent
9. UserManagement
10. AlertEvent
11. DhcpEvent

The `Common` and `AssetEntity` schemas are excluded from tabs (Common fields are included in each schema's field list via the data in asim_fields.csv).

### Lazy Table Initialization

DataTables are initialized only when their containing tab becomes visible. This ensures fast initial page load even with thousands of field rows across all schemas.

### Entity Page Links

Parser names and solution names in the Parsers sub-tab link to their respective documentation pages. The link structure is:

- **Parser pages**: `{docs_base_path}asim/{parser-name}{link_extension}`
- **Solution pages**: `{docs_base_path}solutions/{solution-name}{link_extension}`

When deployed to GitHub Pages alongside the interactive index, set `--docs-base-path "Solutions Docs/"` and `--link-extension ".html"` so that all links resolve correctly.

## Deployment

The ASIM browser is designed to be served from the **same directory** as `index.html` in the sentinelninja GitHub Pages site:

```
sentinelninja/                      # GitHub Pages root
├── index.html                      # Interactive Solutions Analyzer index
├── asim-browser.html               # ASIM Schema Browser
├── css/                            # Shared styles for index.html
├── js/                             # Shared scripts for index.html
└── Solutions Docs/                 # Entity documentation pages
    ├── asim/                       # ASIM parser pages
    ├── solutions/                  # Solution pages
    └── ...
```

## Usage in Pipeline

This script should be run after the input CSVs are generated:

1. **`collect_asim_fields.py`** — Generate `asim_fields.csv`
2. **`map_solutions_connectors_tables.py`** — Generate `asim_parsers.csv`
3. **`generate_asim_browser.py`** — Generate `asim-browser.html`

To regenerate for GitHub Pages deployment:

```bash
python generate_asim_browser.py \
    --output-dir "C:\Users\ofshezaf\GitHub\sentinelninja" \
    --docs-base-path "Solutions Docs/" \
    --index-url "https://oshezaf.github.io/sentinelninja/index.html" \
    --link-extension ".html"
```
