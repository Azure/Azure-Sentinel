📖 How to generate the CSV

1

Configure sentinelEnvironments.json

Create or edit the sentinelEnvironments.json file with the workspaces to analyse:

[ { "subscriptionId": "your-subscription-id", "resourceGroupName": "your-resource-group", "workspaceName": "your-workspace-name" } ]
2

Choose authentication mode

The script supports two authentication modes:

👤 User mode (recommended)

Interactive browser login via device code. No app registration needed — just your user account with Microsoft Sentinel Reader role on the workspace(s).

🔑 App mode

Client credentials (service principal). Requires an App Registration in Entra ID with a client secret and Microsoft Sentinel Reader role assigned.

3

Run the script & load results

Execute the PowerShell script with one of these commands:

# Interactive user login (recommended) .\DefenderAdoptionHelper.ps1 -EnvironmentsFile .\sentinelEnvironments.json -AuthMode User # Service principal (app credentials) .\DefenderAdoptionHelper.ps1 -EnvironmentsFile .\sentinelEnvironments.json -AuthMode App # Omit -AuthMode to be prompted interactively .\DefenderAdoptionHelper.ps1 -EnvironmentsFile .\sentinelEnvironments.json

The script generates a results.csv file. Come back here and drop it on Load CSV Results to see the dashboard.

💡 Note: In User mode, the script opens a device code login — paste the code in your browser and sign in. In App mode, you will be prompted for Tenant ID, Client ID and Client Secret at runtime (no credentials are stored in the script).
